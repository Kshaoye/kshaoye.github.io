多平台镜像构建 ​

1. 概述 ​

一次构建出X86_64及ARM等多CPU指令集容器镜像，满足不同平台的部署需求。

1.1. CPU指令集 ​

  • 当前CPU的两大架构是CISC（复杂指令集）和RISC（精简指令集），x86是CISC的代表架构，占领了95%以上的桌面计算机和服务器市场。
  • Arm和MIPS都是RISC即精简指令集，尤其Arm在智能手机、可穿戴设备(智能手表、手环、智能眼睛)等移动处理器市场占领主要地位。
  • RISC-V和MIPS两大精简指令集架构将会得到广泛使用。

2. 多平台镜像环境准备 ​

2.1. 安装镜像构建环境 ​

可参考基于nerdctl+BuildKit+containerd构建容器镜像，使用Containerd作为镜像构建环境。

2.2. buildkit环境配置 ​

https://github.com/containerd/nerdctl/blob/main/docs/multi-platform.md

2.2.1 配置文件 ​

bash
root@k8s-master1:~# mkdir -p /etc/buildkit
root@k8s-master1:~# vim /etc/buildkit/buildkitd.toml
debug = true
root = "/data/buildkit"
[worker.oci]
enabled = false
[worker.containerd]
address = "/run/containerd/containerd.sock"
enabled = true
platforms = [ "linux/amd64", "linux/arm64" ]
namespace = "buildkit"
gc = true
gckeepstorage = 9000
cniPoolSize = 16
root@k8s-master1:~# systemctl restart buildkit.service

2.2.4 镜像构建 ​

2.2.4.1 Dockerfile ​

dockerfile
#FROM ubuntu:22.04
#FROM registry.cn-hangzhou.aliyuncs.com/myhubregistry/ubuntu:24.04.3-base
FROM registry.cn-hangzhou.aliyuncs.com/myhubregistry/ubuntu:jammy-2424-multi-platform

ADD sources.list /etc/apt/sources.list
ADD ubuntu.sources  /etc/apt/sources.list.d/ubuntu.sources
ARG NGINX_VERSION=1.28.2

# 先替换为 HTTP 源（避免证书问题）
RUN sed -i 's/https:/http:/g' /etc/apt/sources.list && \
    apt-get update && \
    apt-get install -y ca-certificates && \
    update-ca-certificates

RUN sed -i 's/http:/https:/g' /etc/apt/sources.list && \
    apt-get update && \
    apt-get install -y tzdata locales fonts-noto-cjk wget unzip telnet net-tools gcc openssh-server lrzsz tree openssl libssl-dev zlib1g-dev tcpdump traceroute iotop zip make iputils-ping libpcre3 libpcre3-dev iproute2 procps curl lsof less vim && \
    ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

ADD nginx-${NGINX_VERSION}.tar.gz /usr/local/src/
RUN cd /usr/local/src/nginx-${NGINX_VERSION} && ./configure --prefix=/apps/nginx && make && make install  && ln -sv /apps/nginx/sbin/nginx /usr/bin
RUN groupadd  -g 2088 nginx && useradd  -g nginx -s /usr/sbin/nologin -u 2088 nginx && chown -R nginx.nginx /apps/nginx
RUN ln -sf /dev/stdout /apps/nginx/logs/access.log  && ln -sf /dev/stderr /apps/nginx/logs/error.log
RUN echo "alias ll='ls -l'" >> ~/.bashrc && localedef -c -f UTF-8 -i zh_CN zh_CN.utf8 && dpkg-reconfigure -f noninteractive tzdata
ENV LANG=zh_CN.utf8

ADD nginx.conf /apps/nginx/conf/
ADD frontend.tar.gz /apps/nginx/html/
EXPOSE 80 443
ENTRYPOINT ["nginx"]
CMD ["-g","daemon off;"]
#CMD ["nginx","-g","daemon off;"]
